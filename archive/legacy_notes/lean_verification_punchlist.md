## Context (do not edit)

The Lean 4 project at `SemanticConvergence/` (22 modules, 6,520 lines, 270 `theorem` declarations, 231 `def`s, toolchain `leanprover/lean4:v4.29.0`, zero external package dependencies) passes the surface audit:

- 0 `sorry`, 0 `axiom`, 0 `opaque`/`constant`.
- Builds to 23 `.olean` artifacts (in `.lake/build/lib/lean/SemanticConvergence/`).
- `Manifest.lean` closes with `native_decide` proofs establishing `manifestEntryCount = 106`, `derivedEntryCount = 106`, `concreteStackEntryCount = 106`, `fullyFirstPrinciples = true`.

But the adversarial review surfaced three structural problems:

1. **Abstract layer is an assumption package.** `SemanticModel` (Semantic.lean L16–127) encodes every master implication as a user-supplied structure field. `SemanticTheory M` is an empty structure. Abstract theorems are one-line field projections.
2. **Concrete layer has identity-projection proofs.** At least ten theorems in the `ConcretePaperSemantic` namespace have proof bodies that are a single hypothesis name, because the hypothesis unfolds by definitional equality to the conclusion. The flagship `thm_separating_support_convergence` is among them.
3. **Substrate mismatch.** The concrete substrate is `Rat`-valued finite-support laws over a `List BitCode` prefix machine — not the paper's countable universal prefix machine with real-valued probabilities over infinite histories.

The manifest's `fullyFirstPrinciples = true` is a metadata-tag invariant, not a semantic audit: "derived" means a proof term exists, not that the proof term does non-definitional work.

## Tier 1 — Blockers

These are visible to any reviewer who opens the Lean files. Fix first.

### Item 1. Replace `lem_conditional_bc : True` with the real statement

**Where:** `SemanticConvergence/Semantic.lean` line 525–526 (concrete layer) and L29–30, L249–251 (abstract layer field + projection).

**What is broken:** The concrete-layer statement is literally

```
theorem lem_conditional_bc : True := by
  trivial
```

The abstract-layer field `conditionalBorelCantelli : Prop` is an uninterpreted user-supplied predicate, and `conditionalBorelCantelli_holds : conditionalBorelCantelli` is its user-supplied inhabitant.

**Acceptance criterion:** `lem_conditional_bc` has a Prop-valued statement that, on paper, reads as a form of the conditional Borel–Cantelli lemma — namely, under an appropriate filtration, if the sum of conditional probabilities is almost-surely finite then only finitely many of the events occur almost-surely. The Lean statement must be of that shape (quantifier structure: filtration, event sequence, hypothesis on conditional sums, conclusion on finite occurrence) and must be proved. If adding a `require` on `mathlib` is acceptable, the proof can cite `MeasureTheory.MartingaleConvergence` / `ProbabilityTheory.condIndep_borelCantelli` and discharge in roughly a morning. If not, the proof must be built from scratch on the current substrate, which is a multi-week task on its own.

**Decision point for the maintainer:** whether to take a Mathlib dependency. This is the single biggest architectural choice in the punch list; it affects items 2, 3, 4, 7, and 9 simultaneously.

**Effort:** morning (with Mathlib) or month+ (without).

### Item 2. Prove `thm_separating_support_convergence` non-tautologically with the correct conclusion

**Where:** `SemanticConvergence/Semantic.lean` line 546–554 (concrete) and L80–83, L266–270 (abstract field + projection).

**What is broken:** The current concrete statement reads

```
theorem thm_separating_support_convergence
    (U : ConcretePrefixMachine A O) (π : ConcretePolicy A O) (h : FullHist A O)
    (ω : Observer (EncodedProgram A O)) (pstar : EncodedProgram A O)
    (actions : List A) (κ : ActionLaw A)
    (hSupport : def_kernel_sep_condition U π h ω pstar actions κ) :
    ∃ a, a ∈ actions ∧ κ.mass a ≠ 0 ∧ U.isSeparatingAction π h ω pstar a :=
  hSupport
```

The hypothesis `def_kernel_sep_condition` unfolds to `hasSeparatingSupportOn κ actions (U.separatingActionSet π h ω pstar)`, which unfolds to exactly the stated existential. The proof body `hSupport` succeeds only by definitional unfolding. The statement is a notational tautology and, separately, is strictly weaker than the paper's informal claim of posterior concentration.

**Acceptance criterion:** A concrete-layer theorem whose conclusion is a posterior-concentration statement — something of the form

```
∃ T, ∀ (t : Nat), T ≤ t →
  ∀ (hₜ : FullHist A O),  -- constrained to t steps
    U.observerFiberPosteriorMass π hₜ ω pstar ≥ 1 − ε
```

or a clearly finite-time analog (e.g., multiplicative decay per step). The proof must discharge this from a separating-support hypothesis via an actual induction on history length using the Bayesian-update machinery already present in `ConcreteBelief.lean`. Proof body must not be a single hypothesis name, and the conclusion must not unfold definitionally to the hypothesis.

**Dependency:** Uses the per-step decay lemma from item 8.

**Effort:** large.

### Item 3. Fix `thm_summable_support_insufficient` so hypothesis is summability and proof is substantive

**Where:** `SemanticConvergence/Semantic.lean` line 656–660 (concrete) and L48–49, L120–123 (abstract field).

**What is broken:** Current statement:

```
theorem thm_summable_support_insufficient
    (r : Nat → Rat)
    (hEventuallyZero : ∃ N, ∀ n, N ≤ n → r n = 0) :
    ∃ N, ∀ n, N ≤ n → r n = 0 :=
  hEventuallyZero
```

Hypothesis is "eventually zero," which is strictly stronger than summable; conclusion is identical to hypothesis; proof is the identity. The paper's intended statement is the matched-converse sharpness result: summable separating-support schedules are *insufficient* for posterior concentration.

**Acceptance criterion:** The statement has hypothesis `Summable r` (or its concrete analog on `Rat`, e.g. `∃ C : Rat, ∀ N, (Finset.range N).sum (fun n => r n) ≤ C`) and conclusion that exhibits a concrete environment — a specific `ConcretePrefixMachine`, `ConcretePolicy`, and `EncodedProgram` — on which the separating-support schedule is realized by the policy *and* posterior concentration on `[p*]` fails. The existential counterexample should be constructive where possible.

**Effort:** medium.

### Item 4. Fix `cor_support_necessary` so it demonstrates necessity

**Where:** `SemanticConvergence/Semantic.lean` line 644–653.

**What is broken:** Current statement:

```
theorem cor_support_necessary
    ...
    (hNo : ¬ ∃ a, a ∈ actions ∧ κ.mass a ≠ 0 ∧ U.isSeparatingAction π h ω pstar a) :
    ¬ def_kernel_sep_condition U π h ω pstar actions κ :=
  hNo
```

Hypothesis and conclusion are definitionally the same negation. The paper's intended content is: zero separating-support at a history implies posterior concentration on `[p*]` fails at that history.

**Acceptance criterion:** Hypothesis is zero-separating-support on a non-trivial history segment; conclusion is a quantitative failure of posterior concentration (e.g., `U.observerFiberPosteriorMass` is bounded away from `1`, or is bounded by the prior on a fiber that is genuinely not `[p*]`). Proof does real work.

**Effort:** medium.

## Tier 2 — Structural

These change what actually counts as proved and require the Tier 1 machinery.

### Item 5. Audit every concrete-layer theorem for the same projection pattern

**Where:** all of `SemanticConvergence/Concrete*.lean` and the `ConcretePaperSemantic` namespace in `Semantic.lean`.

**What is broken:** At least ten theorems in the `ConcretePaperSemantic` namespace have proof bodies that are a single hypothesis name (see Appendix B of `lean_adversarial_review.md`). The pattern almost certainly recurs in `ConcreteNoise.lean`, `ConcreteSurrogate.lean`, `ConcreteSelfReference.lean`, and `ConcreteRates.lean`. Any theorem with a non-trivial stated implication and a proof body of the form `:= h` or `:= by exact h` is suspect.

**Acceptance criterion:** For every concrete-layer `theorem`, confirm one of: (a) the proof body does non-definitional work (applies a lemma, performs induction, rewrites under a proven equality); (b) the theorem is honestly a definitional unfolding and is re-tagged as such in the manifest (e.g., add a `FormalizationStatus.definitionalUnfold` variant distinct from `derived`); (c) the theorem is rewritten with a non-tautological statement and real proof. Produce a spreadsheet or table listing each theorem, its classification, and the line number.

**Effort:** medium (audit) + large (fixes, distributed across items 2–4 and 7).

### Item 6. Make `SemanticModel` earn its keep or delete it

**Where:** `SemanticConvergence/Semantic.lean` lines 16–127 (structure) and L177 (empty `SemanticTheory`).

**What is broken:** `SemanticModel` is a 110-line structure whose fields include all the master implications as user-supplied proof terms. `SemanticTheory M` is an empty structure, so the abstract "theorems" are trivial field projections of user-supplied proofs.

**Acceptance criterion:** Choose one of the two paths.

- **Path A (keep the abstraction, make it earn its keep):** Construct at least one non-trivial instance `mySemanticModel : SemanticModel` whose implication-fields are *derived* inside the instance construction — i.e., the user does not supply `separatingSupportConvergence` as a black box, but the instance proves it from more primitive fields. The concrete stack's `ConcretePrefixMachine` should be lifted to such an instance. This makes `SemanticTheory M` theorems carry real content because the user cannot construct `M` without discharging the proof obligations.
- **Path B (delete the abstraction):** Remove `SemanticModel`, `SemanticTheory`, and all abstract-layer `theorem` declarations from `Semantic.lean`, `Rates.lean`, `Hierarchy.lean`, `Noise.lean`, `SelfReference.lean`, `Boundary.lean`, `Belief.lean`, `Functional.lean`, `Surrogate.lean`. Retain only the concrete layer. Update the manifest. Update the paper's "layered formalization" narrative to match.

Path B is cleaner and recommended unless the abstract layer serves a clear pedagogical purpose. Path A is what the current file docstrings imply was originally intended.

**Effort:** medium.

### Item 8. Prove at least one genuinely substantive theorem at the substrate level

**Where:** new file, e.g. `SemanticConvergence/ConcretePosteriorDecay.lean`, imported by `ConcreteSemantic.lean` or `Semantic.lean`.

**What is broken:** Nothing in the current concrete substrate does non-trivial probabilistic work. Same-view invariance and finite-time observer refinement (`observerFiberPosteriorMass_eq_of_sameView`, `finiteTimePolicyObserver_refines_policyObserver`) are real but shallow — they are definitional-extensionality lemmas, not probabilistic theorems.

**Acceptance criterion:** Prove a one-step posterior-decay lemma of the following shape, in the current substrate, without adding Mathlib:

> **Lemma.** Let `U : ConcretePrefixMachine A O`, `π : ConcretePolicy A O`, `h : FullHist A O`, `ω : Observer (EncodedProgram A O)`, `pstar : EncodedProgram A O`, `κ : ActionLaw A`. Suppose there exists `a ∈ κ.support` with `κ.mass a ≥ δ > 0` and `U.isSeparatingAction π h ω pstar a`. Let `h'` be the history obtained by sampling action under `κ` and the resulting observation. Then the expected posterior mass on the observer-fiber complement of `[pstar]` satisfies a strict multiplicative decay: `E[ 1 − U.observerFiberPosteriorMass π h' ω pstar | h ] ≤ (1 − c(δ)) · (1 − U.observerFiberPosteriorMass π h ω pstar)` for an explicit computable `c(δ) > 0` (e.g., `c(δ) = δ · bhatSeparation(...)`).

The proof is a finite sum of rationals on the finite machine domain; it does not require measure theory. It does require working out the posterior update formula explicitly. This single lemma is what items 2, 3, 4 each depend on.

**Effort:** medium — and the highest-leverage item on this list. Work this first.

## Tier 3 — Substrate and scope

### Item 9. Resolve the substrate-mismatch honestly

**Where:** `SemanticConvergence/ConcretePrior.lean`, `ConcreteCore.lean` (the `ConcreteLaw`, `ConcretePrefixMachine` definitions); manuscript §§ introducing the universal prior and the interactive prior.

**What is broken:** The paper's object is a countable Turing-complete universal prefix machine, real-valued probabilities, and histories over infinite interaction. The Lean's object is a finite `List BitCode` prefix machine with `Rat`-valued finite-support laws. These are not the same mathematical setting, and the Lean does not prove anything about the paper's setting.

**Acceptance criterion:** Choose one of the two paths.

- **Path A (broaden the Lean):** Add Mathlib. Lift `ConcreteLaw` from `Rat` to `ℝ≥0∞` or `Measure`. Replace `List BitCode` with a countable-enumerable prefix machine (a function `Nat → Option Program` satisfying the prefix-free property). Re-prove everything in items 2, 3, 4, 8 in the broader setting. Months of work.
- **Path B (scope the paper):** In the paper's Lean-parity section, state explicitly that the formalization is of a finite-discrete rational substrate, and that the extension to the universal-prior measure-theoretic setting is an informal argument. Change all rhetorical framing accordingly. Hours of edits.

**Effort:** very large (Path A) or small (Path B).

## Success criterion for "fully verified"

The paper's abstract, introduction, and conclusion can truthfully replace every Lean claim with the **Full** wording from item 15, and a reader running `lake build` followed by inspecting `AxiomAudit.lean` and `Manifest.lean` can independently confirm: (a) the project compiles, (b) every master theorem has a substantive proof with standard-axioms-only dependency, (c) the manifest's `fullyFirstPrinciples = true` is backed by a semantic invariant and not merely a tag count.

Minimum bar for "defensibly verified at the finite-discrete substrate level" is items 1–8, 11–14 done with Path B at items 1 and 9. That is the realistic target for the next model to aim at.
